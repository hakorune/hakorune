# Language Core And Sugar

Status: Historical
Scope: legacy core-plus-sugar design note kept as a redirect stub.

The current Hakorune language direction is not "many sugars lowered to core".
It is "small canonical surface, stronger Stage1 facts, and fail-fast unsupported
rows".

Use these current references instead:

- `docs/development/current/main/design/language-minimal-surface-ssot.md`
- `docs/development/current/main/design/language-feature-implementation-order-ssot.md`
- `docs/reference/language/EBNF.md`
- `docs/reference/language/stage-profiles.md`
- `docs/reference/language/quick-reference.md`

Legacy ideas such as `repeat`, `until`, broad `for`, ternary sugar,
null-coalescing sugar, and safe-access sugar are not canonical current source
surface unless a new accepted row explicitly reopens them.
